💎 核心观点：进化之路

大模型性能的提升，不再仅依赖参数规模。

取决于如何组织与利用上下文信息（Context）。

Context Engineering is the evolution of Prompt Engineering.

⚙️ 何为 Context Engineering

通过结构化地设计、组织、筛选和动态更新上下文信息，以最大化模型的理解与推理能力。

过早信息的遗忘率

（引用 Chroma 等研究）

  • 模型输入窗口有限，过量上下文会导致性能退化。
  • 模型会淡化或忽略过早输入的关键信息 (Context Rot)。
  • 核心挑战：如何在有限窗口内放入“最有价值”的信息。

结论：上下文并非越多越好，而是要“选得准、放得巧、调得动”。

🧩 策略：有效 Context 工程的三层实践

1. System Prompt

Agent 的灵魂 | Prompt Layer

  • 明确身份与语气。
  • 限定任务边界。
  • 给出清晰的输出模板。

2. 即时上下文

临时记忆 | Memory Layer

  • 动态检索历史对话。
  • 嵌入数据库查找相关知识。
  • 基于意图变化自动更新。

目标：每次决策都在“最新、最相关”的语境下。

3. 无限上下文

长期档案 | Architecture Layer

  • 向量数据库 + 检索式记忆。
  • 文档摘要与递归压缩。
  • 分层记忆结构（短期缓存 + 长期档案）。
